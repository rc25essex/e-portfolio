Units 1 - 6 Artefacts

SOLID Principles in Python

9-slide PowerPoint presentation, Python implementation demonstrating SOLID principles and automated unit tests using pytest.

Summary

This artefact builds on concepts introduced in Unit 1 (Object-Oriented Programming) and Unit 2 (SOLID Principles). A presentation was created to explain the core SOLID principles alongside a Python implementation demonstrating how good object-oriented design can improve maintainability, extensibility and code readability. Automated unit tests using pytest were also included to verify behaviour and demonstrate the importance of testable software design.

Learning & Reflection

Key Learning

  • Single Responsibility Principle
  • Open/Closed Principle
  • Dependency Inversion Principle
  • Loosely coupled and testable code
  • Benefits of automated testing

Implementing the example in Python helped me understand how design decisions affect maintainability and future enhancements. The unit tests reinforced the importance of designing classes that are easy to test and extend.

Structural Design Patterns

Seminar presentation and Python implementation demonstrating the Decorator and Composite patterns.

Summary

This artefact builds on Structural Design Pattern concepts introduced in Unit 4. A coffee ordering system was developed to show how objects can be composed and extended without modifying existing code.

Learning & Reflection

Key Learning

  • Decorator Pattern
  • Composite Pattern
  • Structural Design Patterns
  • Object Composition
  • Extensible Software Design

Developing the coffee ordering system improved my understanding of how structural patterns support flexible and maintainable software design through decoration and composition.
