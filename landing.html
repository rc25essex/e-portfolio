SOLID Principles Artefact

Artefacts

  • Python implementation demonstrating SOLID principles
  • Automated unit tests using pytest

Summary

This activity explored the SOLID principles of object-oriented design. I created a presentation explaining each principle and developed a small Python application to demonstrate how SOLID can improve maintainability, extensibility and code readability.

Learning and Reflection

Key Learning

  • Single Responsibility Principle (SRP)
  • Open/Closed Principle (OCP)
  • Dependency Inversion Principle (DIP)
  • Loosely coupled and testable code
  • Benefits of automated testing

Reflection

Prior to this activity I understood object-oriented programming concepts but had limited practical experience applying SOLID principles. Implementing the example in Python helped me understand how design decisions affect maintainability and future enhancements.
