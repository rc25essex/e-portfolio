Capstone Project

Event-Driven Order Processing System

Project Overview

This capstone project demonstrates the design and development of an event-driven order processing system using Java 21, Spring Boot, Apache Kafka and H2.

The application models an Order Service and a Fulfilment Service within a single Spring Boot project. Kafka is used to publish and consume order lifecycle events asynchronously.

Project Objectives

  • Create and store customer orders.
  • Publish order-created events through Kafka.
  • Process orders asynchronously through a fulfilment component.
  • Publish fulfilled or rejected order events.
  • Update and retrieve the current order status.

Technology Stack

  • Java 21
  • Spring Boot
  • Apache Kafka
  • Spring Data JPA
  • H2 in-memory database
  • Gradle
  • JUnit and Mockito
  • Docker Desktop

Architecture Style

The project applies event-driven architecture to separate order creation from fulfilment processing. This reduces direct coupling between the two logical services and allows processing to occur asynchronously.

Event Flow

  1. A client submits an order through the REST API.
  2. The Order Service validates and stores the order.
  3. An OrderCreatedEvent is published to Kafka.
  4. The Fulfilment Service consumes the event.
  5. The order is either fulfilled or rejected.
  6. A result event is published through Kafka.
  7. The Order Service updates the final order status.

Design and Learning

The project was designed incrementally, beginning with requirements, high-level architecture and UML diagrams before implementation. Each application layer was developed and tested before moving to the next stage.

Key Learning

  • Event-driven communication using Kafka producers and consumers.
  • Asynchronous processing and loose coupling.
  • Spring Boot REST and persistence integration.
  • Designing event contracts and order lifecycle states.
  • Incremental development and layered testing.
